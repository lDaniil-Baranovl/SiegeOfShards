# ⚡ Быстрый гайд по обновлению AI системы

## 🎯 Что изменилось

AI теперь поддерживает:
- ✅ **Летающие юниты** - распознаёт и контрит воздушные атаки
- ✅ **3 башни** на каждую команду (как в Clash Royale)
- ✅ **Умная анти-воздушная защита** - приоритизирует юнитов против летающих

## 📋 Чеклист обновления (5 минут)

### ✅ Шаг 1: Настройте карты юнитов

Откройте каждый **UnitCost ScriptableObject** и установите:

**Для наземных без анти-воздуха:**
```
Is Flying: ☐
Can Target Air: ☐
```

**Для наземных с анти-воздухом:**
```
Is Flying: ☐
Can Target Air: ☑   ← ВАЖНО для защиты от летающих!
```

**Для летающих:**
```
Is Flying: ☑
Can Target Air: ☑
```

### ✅ Шаг 2: Обновите BattlefieldAnalyzer

Вместо одной башни теперь список:

**Было:**
```
AI Tower: [одна башня]
Player Tower: [одна башня]
```

**Стало:**
```
AI Towers: [Размер 3]
  Element 0: [AI башня 1]
  Element 1: [AI башня 2]
  Element 2: [AI башня 3]

Player Towers: [Размер 3]
  Element 0: [Player башня 1]
  Element 1: [Player башня 2]
  Element 2: [Player башня 3]
```

### ✅ Шаг 3: Проверьте колоду AI

**ОБЯЗАТЕЛЬНО:** Добавьте минимум 2 карты с `Can Target Air = true`

Рекомендуемая колода (8 карт):
- 2 дешёвых наземных
- **2 анти-воздушных** ← ВАЖНО!
- 2 средних юнита
- 1 тяжёлый юнит
- 1 зелье

### ✅ Шаг 4: Проверьте настройки Health

Убедитесь что у летающих юнитов:
```
Health компонент:
  Can Fly: ☑
```

И у анти-воздушных:
```
UnitStateManager:
  Can Attack Fly Target: ☑
```

### ✅ Шаг 5: Тестирование

1. Запустите игру
2. Разместите летающего юнита
3. В Debug UI должно появиться:
   ```
   Flying: 1
   Flying Threat: YES
   ```
4. AI должен ответить анти-воздушным юнитом
5. В консоли:
   ```
   [AI] Playing: [AntiAirUnit] | Reason: URGENT: Anti-air defense
   ```

## 🐛 Решение проблем

### AI не реагирует на летающих

**Проблема:** Игрок размещает драконов, но AI ставит обычных юнитов

**Решение:**
1. Проверьте `Health.CanFly = true` у дракона
2. Проверьте `UnitCost.isFlying = true` в карте
3. Убедитесь что в колоде AI есть юниты с `canTargetAir = true`
4. Проверьте Debug UI - должно быть "Flying: X"

### AI атакует не ту башню

**Проблема:** AI защищает центральную башню когда атакуют боковую

**Решение:**
1. Убедитесь что все 3 башни добавлены в `aiTowers`
2. Проверьте что башни имеют правильный Team ID
3. Проверьте компонент HealthTower на башнях

### "NullReferenceException" в BattlefieldAnalyzer

**Проблема:** Ошибка при старте игры

**Решение:**
Вы забыли добавить башни! Откройте инспектор BattlefieldAnalyzer:
```
AI Towers: [Размер НЕ должен быть 0]
Player Towers: [Размер НЕ должен быть 0]
```

## 📊 Как проверить что всё работает

### Тест 1: Воздушная атака
```
1. Разместите 2-3 летающих юнита
2. Debug UI показывает "Flying: 2-3"
3. Debug UI показывает "Flying Threat: YES"
4. AI размещает юнита с canTargetAir
5. Console: "URGENT: Anti-air defense"
```

### Тест 2: Множественные башни
```
1. Атакуйте левую башню 3 юнитами
2. Debug UI: "Near Tower: 3"
3. AI защищает именно левую башню
4. Позиция юнита между врагами и левой башней
```

### Тест 3: Смешанная атака
```
1. Разместите 2 летающих + 2 наземных
2. AI ставит сначала анти-воздух (высокий приоритет)
3. Потом ставит наземных защитников
4. Console показывает разные Reason для каждого
```

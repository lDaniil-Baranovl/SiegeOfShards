# 🚁 Обновление AI: Поддержка летающих юнитов и множественных башен

## 📋 Что добавлено

### ✅ Новые возможности

1. **Система летающих юнитов**
   - AI различает летающие и наземные юниты
   - Приоритизирует анти-воздушные юниты против летающих
   - Умная контр-игра против воздушных атак

2. **Множественные башни (как в Clash Royale)**
   - Поддержка 3 башен на каждую команду
   - Защита самой угрожаемой башни
   - Умная расстановка юнитов относительно башен

3. **Анти-воздушная защита**
   - Автоматическое определение воздушной угрозы
   - Приоритет юнитам которые могут атаковать воздух
   - Бонус +70 к приоритету при угрозе с воздуха

## 🔧 Изменения в файлах

### 1. UnitCost.cs - Добавлены свойства

```csharp
[Header("Unit Properties")]
public bool isFlying = false;        // Это летающий юнит
public bool canTargetAir = false;    // Может атаковать летающих
```

**Как настроить карты:**

#### Для наземных юнитов (не могут атаковать воздух):
```
Is Flying: ☐
Can Target Air: ☐
```
Примеры: Скелеты, Центавр, Голем (наземный)

#### Для наземных юнитов (могут атаковать воздух):
```
Is Flying: ☐
Can Target Air: ☑
```
Примеры: Лучник, Стрелки, Башенные юниты

#### Для летающих юнитов:
```
Is Flying: ☑
Can Target Air: ☑ (обычно да)
```
Примеры: Драконы, Летающие маги

### 2. BattlefieldAnalyzer.cs - Множественные башни

**Новые поля:**
```csharp
[SerializeField] private List<Transform> aiTowers = new List<Transform>();
[SerializeField] private List<Transform> playerTowers = new List<Transform>();
```

**Новая информация в BattlefieldState:**
```csharp
public int playerFlyingCount;      // Сколько летающих юнитов у игрока
public int playerGroundCount;      // Сколько наземных юнитов у игрока
public Transform mostThreatenedTower;  // Какая башня под наибольшей угрозой
public bool hasFlyingThreat;       // Есть ли угроза с воздуха
```

**Новые методы:**
- `GetMostThreatenedTower()` - находит башню с наибольшим количеством врагов рядом
- `GetClosestAliveTower()` - находит ближайшую живую башню
- `GetPlayerFlyingUnits()` - список летающих юнитов игрока
- `GetPlayerGroundUnits()` - список наземных юнитов игрока

---

### 3. AICardSelector.cs - Умная контр-игра

**Новый параметр:**
```csharp
[SerializeField] private float antiAirPriorityBonus = 30f;  // Бонус приоритета для анти-воздушных юнитов
```

**Логика выбора:**

Когда игрок использует летающих юнитов:
```
1. AI обнаруживает: hasFlyingThreat = true
2. AI ищет юнитов с canTargetAir = true
3. Добавляет +30 базовый бонус
4. При защите: дополнительно +40
5. Итого: приоритет 120+ для анти-воздушных юнитов
```

**Пример приоритетов:**

| Сценарий | Обычный юнит | Анти-воздушный юнит |
|----------|--------------|---------------------|
| Нет летающих врагов | 70 | 70 |
| Есть летающие враги | 70 | 140 (70 + 30 + 40) |
| Критическая угроза с воздуха | 105 | 210 |

## 🎮 Примеры работы новой системы

### Сценарий 1: Воздушная атака игрока

```
1. Игрок размещает 3 летающих дракона
2. Analyzer: playerFlyingCount = 3, hasFlyingThreat = true
3. CardSelector ищет юнитов с canTargetAir = true
4. Лучники: приоритет 70 + 30 + 40 = 140
5. Обычные скелеты: приоритет 70
6. AI выбирает Лучников
7. Логирование: [AI] Strategy: Defend | Playing: Archer |
   Reason: URGENT: Anti-air defense | Priority: 140.0
```

### Сценарий 2: Атака на разные башни

```
1. Игрок атакует левую башню (3 юнита рядом)
2. У правой башни 1 юнит
3. GetMostThreatenedTower() возвращает левую башню
4. AI размещает защитников у левой башни
5. Позиция: между врагами и левой башней
```

### Сценарий 3: Смешанная атака

```
1. Игрок: 2 летающих + 3 наземных
2. AI анализирует:
   - playerFlyingCount = 2
   - playerGroundCount = 3
   - hasFlyingThreat = true
3. AI выбирает стратегию:
   - Сначала ставит анти-воздушного (приоритет 140)
   - Потом дешёвых защитников против наземных
```

## ⚙️ Настройка в Unity

### Шаг 1: Настройте карты юнитов

Для каждого **UnitCost ScriptableObject**:

#### Наземные атакеры (только земля):
```
Card Type: Unit
Is Flying: ☐
Can Target Air: ☐
```
Примеры: Скелет, Варвар, Голем

#### Наземные с анти-воздухом:
```
Card Type: Unit
Is Flying: ☐
Can Target Air: ☑
```
Примеры: Лучник, Стрелки, Маг

#### Летающие юниты:
```
Card Type: Unit
Is Flying: ☑
Can Target Air: ☑ (обычно)
```
Примеры: Дракон, Летающий маг, Минион

### Шаг 2: Настройте BattlefieldAnalyzer

В инспекторе:

```
AI Towers: [Размер 3]
  Element 0: AI Tower Left
  Element 1: AI Tower Center
  Element 2: AI Tower Right

Player Towers: [Размер 3]
  Element 0: Player Tower Left
  Element 1: Player Tower Center
  Element 2: Player Tower Right

AI Team ID: 1
Player Team ID: 0
```

### Шаг 3: Составьте сбалансированную колоду AI

**Рекомендуемый состав (8 карт):**

1. **2 дешёвых наземных** (1-2 элексира)
   - Can Target Air: ☐
   - Для быстрой защиты

2. **2 анти-воздушных** (2-4 элексира)
   - Can Target Air: ☑
   - ОБЯЗАТЕЛЬНО для контр-игры

3. **2 средних юнита** (4-6 элексира)
   - Смесь: 1 с анти-воздухом, 1 без

4. **1 тяжёлый юнит** (6-8 элексира)
   - Танк или мощный атакер

5. **1 зелье** (3-5 элексира)
   - Freeze, Meteor или Heal

**⚠️ Важно:** Минимум 2-3 юнита с `Can Target Air = true` для защиты от воздуха!

## ❓ FAQ

**Q: AI не реагирует на летающих юнитов**

Проверьте:
1. У летающих юнитов `Health.CanFly = true`
2. У карты `UnitCost.isFlying = true`
3. В колоде AI есть юниты с `canTargetAir = true`
4. В Debug UI видно "Flying: X"

---

**Q: AI ставит не тех юнитов против летающих**

Проверьте:
1. У анти-воздушных карт `canTargetAir = true`
2. `antiAirPriorityBonus` >= 30 в AICardSelector
3. В колоде минимум 2 анти-воздушных юнита

---

**Q: AI защищает не ту башню**

Проверьте:
1. Все 3 башни добавлены в `aiTowers`
2. У башен корректный teamID
3. HealthTower компонент работает корректно

---

**Q: Как настроить юнита который может атаковать всех?**

```
Is Flying: ☐  (если наземный)
Can Target Air: ☑  (может атаковать воздух)
```

В UnitStateManager:
```
canAttackFlyTarget: true
```

---

**Q: Летающий юнит может атаковать других летающих?**

Обычно да:
```
Is Flying: ☑
Can Target Air: ☑
```
